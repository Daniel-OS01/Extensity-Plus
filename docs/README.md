# Docs

This folder captures the Extensity 2.0 rollout from three angles:

- `extensity-2.0-plan.md`: the implementation plan, architectural decisions, KSB constraints, and design patterns used across all rounds.
- `extensity-2.0-status.md`: what was actually implemented across Rounds 1, 2, and 3, including gaps and deferred work.
- `ci-and-release.md`: how CI, build packaging, and Chrome Web Store submission artifacts work, plus the full manual test checklist.
- `chrome-web-store-checklist.md`: the in-repo manifest/icon checks and the remaining Chrome Web Store dashboard fields that must still be filled manually.

For the active implementation plan (Round 3 pending changes), see `.claude/plans/zazzy-hugging-pearl.md`.

For the high-level product overview, see the repository root `README.md`.
For local build steps, see `BUILD.md`.
